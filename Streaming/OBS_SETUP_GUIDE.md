## 🔑 Как получить Stream Key

### Способ 1: Через API
```bash
# Получить список активных ключей
curl http://YOUR_SERVER_IP:8080/api/streaming/keys

# Создать новый ключ
curl -X POST http://YOUR_SERVER_IP:8080/api/streaming/keys \
  -H "Content-Type: application/json" \
  -d '{"name": "OBS Stream", "description": "Main streaming key"}'
```

### Способ 2: Через Web интерфейс
1. Открыть http://YOUR_SERVER_IP:8080 (или https://footballteams.ru)
2. Перейти в раздел "Настройки"
3. Скопировать Stream Key или создать новый

### Способ 3: Использовать тестовый ключ
```
Stream Key: test-stream-key
```
*(по умолчанию создается тестовый ключ для быстрого старта)*

## 🛠️ Пошаговая настройка OBS

### Шаг 1: Установка OBS Studio
1. Скачать OBS Studio с официального сайта: https://obsproject.com/
2. Установить последнюю версию (рекомендуется 30.0+)

### Шаг 2: Базовые настройки

#### **Настройки → Вывод:**
```
Режим вывода: Расширенный

=== Потоковая передача ===
Аудиокодек: AAC
Управление битрейтом: CBR
Битрейт: 3000-6000 Kbps (в зависимости от качества интернета)
Интервал ключевых кадров: 2
Профиль ЦП: veryfast (или fast для лучшего качества)
Tune: zerolatency

=== Запись ===
Тип: Стандартный
Формат записи: mp4
Кодировщик: x264 (или аппаратный если доступен)
```

#### **Настройки → Видео:**
```
Базовое разрешение: 1920x1080
Выходное разрешение: 1920x1080 (или 1280x720 для слабых ПК)
Фильтр понижения разрешения: Бикубический
FPS: 30 (или 60 для игр)
```

#### **Настройки → Звук:**
```
Частота дискретизации: 48 кГц
Канала: Стерео

Устройство воспроизведения по умолчанию: Ваши динамики/наушники
Устройство записи по умолчанию: Ваш микрофон
```

### Шаг 3: Настройка потока

#### **Настройки → Трансляция:**

**Для локальной разработки:**
```
Сервис: Пользовательский...
Сервер: rtmp://localhost:1935/live
Ключ потока: test-stream-key
```

**Для FirstVDS (IP):**
```

Сервер: rtmp://62.109.28.207:1935/live  # Замените на ваш IP
Ключ потока: your-actual-stream-key
```

**Для FirstVDS (домен):**
```
Сервис: Пользовательский...
Сервер: rtmp://footballteams.ru:1935/live
Ключ потока: your-actual-stream-key
```

## 🧪 Проверка и тестирование

### 1. Проверить подключение к серверу:
```bash
# Проверить доступность RTMP порта
telnet YOUR_SERVER_IP 1935

# Или через PowerShell
Test-NetConnection YOUR_SERVER_IP -Port 1935
```

### 2. Тестовый стрим:
1. Запустить стрим в OBS (кнопка "Начать трансляцию")
2. Проверить статус на сервере:
   - Web: http://YOUR_SERVER_IP:8080
   - API: http://YOUR_SERVER_IP:8080/api/streaming/status
3. Посмотреть плейбек: http://YOUR_SERVER_IP:8081/live/your-stream-key.m3u8

### 3. Проверить качество:
```
- Стабильность битрейта (зеленый индикатор в OBS)
- Отсутствие пропущенных кадров  
- Задержка стрима (обычно 3-10 секунд)
- Качество видео и звука
```

## 🔧 Решение проблем

### ❌ "Не удается подключиться к серверу"

**Причины и решения:**
```
1. Неправильный URL сервера
   → Проверить IP/домен и порт 1935

2. Файрвол блокирует соединение
   → ufw allow 1935/tcp на сервере

3. RTMP сервер не запущен
   → docker-compose ps | grep rtmp

4. Неправильный Stream Key
   → Проверить ключ через API или web
```

### ❌ "Подключается, но стрим не отображается"

**Решения:**
```
1. Проверить логи RTMP сервера:
   docker-compose logs rtmp-server

2. Проверить API логи:  
   docker-compose logs api

3. Убедиться что Stream Key валидный:
   curl http://localhost:8080/api/streaming/keys

4. Проверить статус стрима:
   curl http://localhost:8080/api/streaming/status/your-stream-key
```

### ❌ "Высокая задержка или лаги"

**Оптимизации:**
```
1. Снизить битрейт (2000-4000 Kbps)
2. Использовать Tune: zerolatency
3. Уменьшить интервал ключевых кадров (1-2)
4. Проверить пинг до сервера: ping YOUR_SERVER_IP
5. Использовать проводное подключение вместо WiFi
```

### ❌ "OBS тормозит/лагает"

**Решения:**
```
1. Снизить разрешение до 720p
2. Уменьшить FPS до 30
3. Использовать ultrafast профиль ЦП
4. Включить аппаратное ускорение
5. Закрыть ненужные программы
6. Отключить предпросмотр в OBS
```

## 📊 Мониторинг стрима

### В OBS:
- **Зеленый индикатор** → все хорошо
- **Желтый** → небольшие проблемы
- **Красный** → серьезные проблемы

### Статистика (Вид → Статистика):
```
Пропущенные кадры: < 1%
Загрузка ЦП: < 80%
Битрейт: стабильный
Время стрима: отслеживание продолжительности
```

### На сервере:
```bash
# Мониторинг ресурсов
docker stats --no-stream

# Логи стримов
tail -f /opt/streaming/media/logs/*.log

# Активные стримы
curl http://localhost:8080/api/streaming/active
```
